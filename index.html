Abstract

Quantum-Resistant Authentication

Replacing classical JWT signing algorithms with a lattice-based post-quantum alternative across a four-component microservice architecture.

JSON Web Tokens are the de facto standard for stateless authentication in distributed systems. The classical signing algorithms they rely upon — RSA and ECDSA — are mathematically vulnerable to quantum computing attacks via Shor's algorithm, threatening the security of virtually all modern web authentication infrastructure.

This research designs, implements, and evaluates a Post-Quantum JWT Signing and Verification Service (P1) using ML-DSA-44 (CRYSTALS-Dilithium), standardised in NIST FIPS 204. The service preserves full JOSE compatibility — RFC 7515 JWS serialisation, kid-based key discovery, structured error taxonomy — while replacing the signing primitive with a lattice-based algorithm resistant to quantum attacks.

The framework is deployed as four tightly integrated microservices: P1 (signing), P2 (JWKS & transparency), P3 (guard & policy middleware), and P4 (revocation). Evaluation confirms post-quantum JWT authentication is operationally viable with competitive latency and zero request failures under realistic load.

Key Research Findings

Benchmarked against an RS256 classical baseline across latency, token size, stability, and key management.

Latency

Competitive Tail Latency

PQC p95 login (102.5 ms) and p99 (115.8 ms) are both lower than RS256 baseline p95 (117.4 ms) and p99 (206.0 ms). The algorithm is not the bottleneck.

Sub-Millisecond Operations

ML-DSA-44 signing completes in 0.148–0.367 ms and verification in 0.008–0.095 ms. The cryptographic overhead is negligible relative to network latency.

Domain

Research Domain

The problem space, identified research gaps, and scientific contributions of this work.

01 · Research Gap

Missing Production PQC JWT

No major identity provider (Auth0, Keycloak) supports ML-DSA natively. Prior work lacks JOSE compliance, key management, and tail-latency benchmarking at system level.

02 · Research Problem

Quantum Threat to JWT Auth

RS256 and ES256 are broken by Shor's algorithm. The "harvest now, decrypt later" strategy motivates migration to post-quantum algorithms before quantum computers become available.

03 · Methodology

Design, Build, Evaluate

Implement the P1 signing service, integrate with three companion components, and benchmark against an RS256 baseline capturing average, p95, p99 latency and zero-failure stability.

04 · Contribution

JOSE-Compatible ML-DSA Service

First production-oriented, Dockerised ML-DSA-44 JWT signing and verification service with deterministic key identifiers, multi-active key management, and Prometheus observability.

05 · Literature

Built on NIST PQC Foundations

Grounded in NIST FIPS 204, RFC 7515–7519, and feasibility studies by Alkhulaifi & El-Alfy (2020) and Dhanush & Jain (2023). Advances prior art from benchmarks to a complete integration architecture.

06 · Impact

Quantum-Ready Before 2030

Demonstrates organisations can migrate JWT authentication to post-quantum cryptography today within existing microservice architectures — ahead of NIST's 2030 migration deadline.
